# Vision

## The problem we're solving

Developers spend a large portion of their time writing predictable, repetitive code: CRUD endpoints, database schemas, validation logic, test boilerplate. This is necessary work but not the interesting part of building software.

LLMs are already good at generating this kind of code. The gap is tooling: most AI coding tools are either editor plugins (Copilot, Cursor) or general chatbots. There's no good tool that:

1. Runs specialized bots locally (privacy)
2. Coordinates multiple bots for a task (orchestration)
3. Gets cheaper as more people use it (network effects pricing)

That's what Resonance aims to be.

---

## What success looks like

A developer opens Resonance, describes what they need to build, and gets working boilerplate in seconds — not a chat response, actual files ready to run and modify.

They trust it because:
- Their code never left their machine
- They ran the benchmarks and saw the real numbers
- The price is fair and gets fairer over time

---

## What we're not trying to be

We are not trying to replace developers or create artificial general intelligence. The goal is narrower and more achievable: make the boring parts of software development fast so developers can spend time on the parts that require genuine judgment.

---

## On the name "Resonance"

The name is kept because it's good branding. It does not imply any physical resonance mechanism. The technical reality is: parallel LLM calls with specialized prompts, combined into structured output. That's it, and that's enough.
